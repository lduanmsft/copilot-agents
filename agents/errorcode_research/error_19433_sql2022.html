void
CHadrArProxy::ComputeInitialStateInWsfc ()
{
    AUTO_XE_FOR_HADR_API (m_agName);

    // WsfcIsAgIntactInWsfc() in AGOM is very optimistic.  If any exception is
    // caught, and the exception does not correspond to object-not-found, the
    // method assumes that all AG objects exist in WSFC and will return 'true'.
    //
    // The contract: this call will never return false negative, but may
    // return false positive.  If it is indeed false positive ('true'), then
    // the next startup of this AR will do a local drop-AG, which is correct.

    EX_TRY_SILENT
    {
        m_fAgInWsfc =
                m_pAgom->WsfcIsAgIntactInWsfc (
                    m_agName,
                    m_agGroupId,
                    m_agResId);
    }
    EX_CATCHSE (se)
    {
        if (ERR_ENUM_CLUSTER_REG_VALUE == se.Minor () &&
            ERROR_CLUSTER_REG_ENUM_VALUE_API_FOR_AG_CONFIG_READ_EXHAUSTED ==
                se.State ())
        {
            DELAYABORT_IN_THIS_SCOPE
            EnqueueRestartNotification ();
        }
    }
    EX_CATCH_END;
}

Step 8 Function Logic & Analysis

🧠 WsfcIsAgIntactInWsfc() — WSFC Cluster Integrity Check

Performs a 4-step validation of AG objects in the Windows Server Failover Cluster:

  • Step 1 — AG Name→ID Map: Calls RetrieveAgNameToIdMapEntry(), then compares hadrAgGroupId with the retrieved group ID. ⚠ Error 19433 fires here when they don't match.
  • Step 2 — Resource ID Check: Compares hadrAgResId similarly (fires error 19434 on mismatch).
  • Step 3 — Cluster Resource Existence: Opens the cluster resource handle via GetClusterResourceByIdOrName().
  • Step 4 — SQL Instance→Node Map: For WSFC AGs, verifies the local instance has an entry in the cluster registry.

⚙️ Error Handling Behavior

  • Raised via: scierrlog — non-fatal, logged to SQL error log only, no exception thrown
  • Returns: false immediately — AG is considered not intact in WSFC
  • Exception handling: EX_TRY_SILENT / EX_CATCHSE wraps the whole function:
    • Specific not-found exceptions → retVal = false
    • Other exceptions → optimistically returns true (assumes WSFC quorum loss)
    • ERR_ENUM_CLUSTER_REG_VALUE with exhausted state → enqueues restart notification
  • Root cause: AG group ID stored in WSFC cluster registry has diverged from the local SQL Server metadata — can happen after cluster repair, manual registry edits, or AG reconfiguration failures.
