📚 Related Microsoft Learn Docs

Cross-Database Queries (In-Memory OLTP)

Section: Main article

Starting with SQL Server 2014 (12.x), memory-optimized tables do not support cross-database transactions. You cannot access another database from the same transaction or the same query that also accesses a memory-optimized table.
Table variables are not transactional. Therefore, memory-optimized table variables can be used in cross-database queries, and can thus facilitate moving data from one database into memory-optimized tables in another. You can use two transactions.

🛠️ Workarounds

Workaround 1: Use memory-optimized table variables for cross-database data transfer

Source: Microsoft Learn — Cross-Database Queries, Section: main article

Table variables are not transactional. Therefore, memory-optimized table variables can be used in cross-database queries.
-- Use two separate transactions
DECLARE @tmp dbo.MemoryType;
INSERT @tmp SELECT * FROM SourceDB.dbo.SourceTable;  -- TX 1
INSERT dbo.DestTable_InMem SELECT * FROM @tmp;       -- TX 2
